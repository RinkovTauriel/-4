**Алгоритм (пошагово)**

 Шаг 1: Построение минимального остовного дерева (MST)

1.1. Инициализируем массив посещенных вершин и минимальных весов

1.2. Начинаем с вершины 0, добавляем в MST

1.3. На каждой итерации выбираем ребро минимального веса, соединяющее MST с новой вершиной

1.4. Добавляем вершину в MST и обновляем минимальные веса

1.5. Повторяем до включения всех вершин

 Шаг 2: Удвоение ребер и построение эйлерова графа

2.1. Для каждого ребра (u, v) в MST добавляем его копию

2.2. Получаем эйлеров граф (все вершины имеют четную степень)

2.3. Строим мультиграф с повторяющимися ребрами

 Шаг 3: Поиск эйлерова цикла

3.1. Используем алгоритм Флёри или итеративный DFS

3.2. Начинаем с произвольной вершины (обычно 0)

3.3. На каждом шаге выбираем ребро, не являющееся мостом (если возможно)

3.4. Удаляем пройденные ребра из графа

3.5. Продолжаем, пока не пройдем все ребра

 Шаг 4: Построение гамильтонова цикла

4.1. Обходим эйлеров цикл, отслеживая посещенные вершины

4.2. При встрече с уже посещенной вершиной - пропускаем ее

4.3. Добавляем только первые вхождения каждой вершины

4.4. В конце добавляем стартовую вершину для замыкания цикла

 **Временная сложность**
 Временная сложность - это оценка времени работы алгоритма в зависимости от размера входных данных.

· Построение MST: O(E log V) = O(n² log n) для полного графа

· Поиск эйлерова обхода: O(E) = O(n)

· Построение гамильтонова цикла: O(n)

· Итоговая сложность: O(n² log n)


 Для данного алгоритма TSP:

O(n²) - квадратичная сложность

· n - количество городов

· n² операций - при увеличении n в 2 раза, время работы увеличивается в 4 раза

 Разбивка по шагам:

1. MST (Прим): O(n²) - основной вклад
2. Эйлеров обход: O(n) - незначительный
3. Гамильтонов цикл: O(n) - незначительный

 Практические значения:

· 10 городов: ~100 операций

· 50 городов: ~2,500 операций

· 100 городов: ~10,000 операций


 **Коэффициент аппроксимации**
 
Аппроксимация - гарантия того, что найденное решение не более чем в K раз хуже оптимального.
 Определения:
· OPT - стоимость оптимального тура

· MST - стоимость минимального остовного дерева

· APPROX - стоимость найденного тура


 Алгоритм гарантирует 2-аппроксимацию:

· Вес MST ≤ вес оптимального TSP-тура

· Вес эйлерова обхода = 2 × вес MST

· После shortcuts вес не увеличивается
Итог: APPROX ≤ 2×OPT


 Для данного алгоритма TSP:

· Гарантия: стоимость найденного тура ≤ 2 × стоимость оптимального тура

· На практике: обычно 1.3-1.8 × оптимального

 Условия применения:

· Метрический TSP: расстояния удовлетворяют неравенству треугольника

· Симметричный TSP: dist(u,v) = dist(v,u)

· Полный граф: все города соединены между собой

Контрольный вопрос 6

В чем заключается принципиальное различие между дробным рюкзаком и 0-1 рюкзаком?

Ответ:

Дробный рюкзак (Fractional Knapsack):

· Предметы можно дробить на части

· Можно взять часть предмета (например, 0.5 от веса)

· Решается оптимально жадным алгоритмом за O(n log n)

· Коэффициент аппроксимации: 1 (точное решение)

0-1 рюкзак (0-1 Knapsack):

· Предметы нельзя дробить

· Можно либо взять предмет целиком, либо не брать

· NP-сложная задача

· Жадный алгоритм не гарантирует оптимальность

· Требует динамического программирования или приближенных алгоритмов

Ключевое отличие: Возможность дробления предметов делает дробный рюкзак значительно более простой задачей, решаемой за полиномиальное время, в то время как 0-1 рюкзак является NP-сложной задачей.
